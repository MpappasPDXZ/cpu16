    \item \textbf{Reduce Complexity:} Implement orchestration (Dagster) and publish/subscribe messaging to eliminate redundancy

    \item \textbf{All Data Types:} Unify structured data (rows/columns) with unstructured data (documents, models, schedules) in a single pipeline
\end{enumerate}
}

\vspace{0.1em}

{\footnotesize
\textbf{1. Events (Pub/Sub):} Message bus for event-driven architecture
\begin{itemize}[leftmargin=1em, itemsep=0em]
    \item \textbf{Two Aspects:} Message queuing (Azure Event Grid) vs. data streaming (where data is the message)
    \item \textbf{Recommendation:} Confluent Kafka (Apache Kafka-based streaming + messaging)
    \item \textbf{Alternative:} Azure Event Hub (streaming) + Event Grid (messaging)
    \item \textbf{Confluent advantage:} Flink integration, catalog connectors, avoids hyperscaler lock-in
    \item \textbf{Status:} Met with Confluent on January 16, 2026; awaiting NDA signature and pricing
    \item \textbf{Implementation:} Confluent recommended vendor 'Improving' for expertise in large data streaming applications
\end{itemize}

\vspace{0.15em}

\textbf{2. Orchestrator (Dagster+):} Central hub of KIP
\begin{itemize}[leftmargin=1em, itemsep=0em]
    \item \textbf{Hosting:} GKE installed, while we continue to research AKS in parallel for secondary deployment
    \item 2-year contract with token rollover; awaiting NDA (01.06)
    \item Hybrid model: Dagster+ on GKE, Azure, and on-prem cluster
    \item Assets shared across ecosystem (single pane of glass)
\end{itemize}

\vspace{0.15em}

\textbf{3. Code Management:} GitHub $\rightarrow$ Dagster environment

\vspace{0.15em}

\textbf{4. Compute:} Kubernetes compute pools (standard)
\begin{itemize}[leftmargin=1em, itemsep=0em]
    \item GCP: BigQuery (distributed compute over Iceberg)
    \item Azure: Databricks or Trino (self-managed)---Microsoft lacks native Iceberg compute
\end{itemize}

\vspace{0.15em}

\textbf{5. Storage:} Iceberg medallion architecture
\begin{itemize}[leftmargin=1em, itemsep=0em]
    \item \textbf{Bronze:} Raw parquet (virtualized to Snowflake via catalog)
    \item \textbf{Silver Product:} Iceberg + parquet (API endpoints may read directly)
    \item \textbf{Silver Domain:} Topically-organized Iceberg tables
    \item \textbf{Gold:} Iceberg tables for analytics
    \item \textbf{Self-Service:} Snowflake (business units create tables, join to governed KIP data)
    \item \textbf{Note:} This is an executive summary but actual medallion will have more layers.
\end{itemize}

\vspace{0.15em}

\textbf{6. Data Science Services:}
\begin{itemize}[leftmargin=1em, itemsep=0em]
    \item Closed-source models, vector databases
    \item BigLake catalog, BigQuery (where it makes sense to use)
    \item GCP preferred for AI model synergy with the data science data structures
    \item GCP has been used for the past few years for Gemini models due to the performance and efficiency.
    \item Modal is used for model inference in some cases to reduce complexity of model development and deployment.
\end{itemize}
}
