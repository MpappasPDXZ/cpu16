================================================================================
                    KEY INSIGHTS & SUPPORTING EVIDENCE
================================================================================

1. InEight User Base:
   • 3,127 execution staff use InEight (26% of execution workforce)
   • Field Engineers are largest user group (1,688, 54% of InEight users)
   • Superintendents second largest (1,356, 43% of InEight users)
   • Project Engineers smallest group (83, 3% of InEight users)
   
   Supporting Evidence: Cumberland project shows 52.3% of staff are InEight users
   when including all field operations roles. After overhead adjustment, this 
   aligns with 26% of execution staff.

2. Module Usage Patterns:
   • Inspect module: Highest usage (29.4%) - quality/safety focus
   • Design module: Second highest (26.6%) - engineering workflows
   • Contract module: Core functionality (20.3%) - project management
   • Core module: Base platform (19.6%) - system foundation
   • Control module: Lower usage (4.2%) - specialized function
   • Billings/Plan: Not used in Cumberland project
   
   Supporting Evidence: Screen time analysis from Cumberland project (105778)
   shows actual distribution: Contract (29), Control (6), Core (28), 
   Design (38), Inspect (42), Billings (0), Plan (0).

3. Time Investment:
   • 1.6M hours/year across all InEight users
   • 520 hours/person/year average (25% of work time)
   • Field Engineers: 877K hours/year (54% of total)
   • Superintendents: 705K hours/year (43% of total)
   
   Supporting Evidence: Based on 2,080 hours/year standard work year × 25% 
   usage assumption (validated against typical construction software usage).

4. Overhead Impact:
   • 20% of salaried staff in overhead roles (district + corporate)
   • Execution staff: 11,958 (80% of salaried)
   • District overhead: 1,494 (10%) - estimating, district management
   • Corporate overhead: 1,494 (10%) - home office, shared services
   
   Supporting Evidence: Industry standard for construction companies shows
   10-15% district overhead and 5-10% corporate overhead. 10% each is 
   conservative estimate.

================================================================================
                    DATA SOURCES & METHODOLOGY
================================================================================

Primary Data Sources:
  1. employee_data_cumberland.xlsx - 216 active staff records
  2. cumberland_data_2.xlsx - 149 staff records (verification)
  3. Cumberland project screen usage data (screenshot analysis)
  4. Kiewit enterprise headcount: 14,946 salaried staff

Methodology:
  1. Job title categorization using enhanced matching (errs on inclusion)
  2. Cumberland project percentages applied as proxy to Kiewit execution staff
  3. Overhead adjustment: 20% removed (10% district + 10% corporate)
  4. Hours calculation: 2,080 hours/year × 25% InEight usage
  5. Module distribution: Based on actual Cumberland screen time data

Limitations:
  • Cumberland is single project - may not represent all project types
  • Project Engineer count may be low due to project phase/structure
  • Screen usage from one project snapshot - may vary by project type
  • 25% usage assumption based on industry standards, not measured
