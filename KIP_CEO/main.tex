 \begin{center}
     \noindent\rule{\linewidth}{2pt}
     
    \vspace{0.15em}
    
    {\Large What is KIP?} \\[0.2em]
    {\small A four-layer platform integrating enterprise data sources, modern data factory capabilities, AI-powered intelligence, and operational automations}
    
    \vspace{0.15em}
     
     \noindent\rule{\linewidth}{2pt}
\end{center}

\vspace{0.1em}

% Two-column layout: levels on left, image and pain points on right
 {\footnotesize
\begin{tabular}{@{}p{0.25\textwidth}@{\hspace{0.05\textwidth}}p{0.70\textwidth}@{}}
\raggedright\arraybackslash
\vspace{0.15em}

{\footnotesize \textbf{L0: Data Sources}

{\scriptsize Enterprise systems (SAP, InEight, GEP), P6 schedules, estimate files, 3rd party contracts, engineering models, procurement data (Source360), spatial scans and video capture (FieldAI), SharePoint, and real-time operational streams}}

\vspace{0.4em}

{\footnotesize \textbf{L1: Data Factory}

{\scriptsize Open-source orchestration platform with modern lakehouse architecture. Ingests enterprise data from SAP, InEight, and Primavera to data lake. Model Context Protocol (MCP) servers provide LLMs secure, standardized access to data assets with lineage tracking}}

\vspace{0.4em}

    {\footnotesize \textbf{L2: Kiewit Intelligence}
    
    {\scriptsize AI-powered data ingestion to open-source tables via Polaris catalog. 20,000+ predictive algorithms (FastAPI/Azure) deliver industry-leading labor hour estimates. AI agents for conversational search and document generation. Centralized data science team drives continuous quality improvement at ingestion}}

\vspace{0.4em}

    {\footnotesize \textbf{L3: Automations}
    
    {\scriptsize Operational workflow automation for Takeoffs, Estimating, Scheduling, and Work Planning. Powered by 16-node NVIDIA B200 high-performance compute cluster. Data science team ramping to deliver operational staff improvements and back-office reductions}}
 &
     \begin{minipage}[t]{\linewidth}
         \raggedleft
         \raisebox{-\height}{\includegraphics[width=0.90\textwidth]{landscape.png}}
         
         \vspace{0.6em}
         
         \raggedright
        \begin{tikzpicture}
            \node[fill=black!10, rounded corners=3pt, inner sep=4pt, text width=0.95\linewidth, align=left] {\scriptsize\textbf{Addressed Data Science Velocity Hindrances}};
        \end{tikzpicture}
         
         \vspace{0.2em}
         
         {\scriptsize
    {\footnotesize \textbf{1. Data Access \& Discoverability:}} Controlled access to medallion storage requires separate data engineering involvement. Open-source table technology makes operational data (model quantities, component claiming, scheduling) discoverable in both data lake and warehouse.
    
    \vspace{0.15em}
    
    {\footnotesize \textbf{2. Data Availability:}} Design model data not ingested/normalized, InEight scheduling relationships unavailable, custom systems (LAND, KADE, ADAPT) need integration. Partnering for high-velocity ingestion of key sources like Source360.
    
    \vspace{0.15em}
    
    {\footnotesize \textbf{3. Orchestration \& Redundancy:}} Critical capability to connect pipelines and monitor data health. Enable reverse engineering to eliminate dataset duplication. Reduce enterprise redundancy (36+ Snowflake databases plus on-premises ODS).
    
    \vspace{0.15em}
    
    {\footnotesize \textbf{4. Skills \& Organization:}} Build internal data engineering capabilities in single organization with joint priorities. Augment with external partners (phData, InCycle) while developing key domain expertise internally.
         }
     \end{minipage}
     \end{tabular}
     }
